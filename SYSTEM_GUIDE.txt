# 🚁 Rescue System Ecosystem - Technical Guide & Credentials

This guide provides everything you need to run, test, and manage the Rescue System Ecosystem.

---

## 🛠️ Tech Stack
- **Backend**: Node.js with Express
- **Database**: SQLite3 (Local file-based database)
- **Real-time**: WebSockets (WS) for live dispatch and status updates
- **Frontend**: Vanilla HTML5, CSS3 (Modern Light Themes), and Javascript
- **Maps**: Leaflet.js with OpenStreetMap/CartoDB tiles

---

## 🚀 How to Run Locally (Dynamic Mode)

To ensure the apps are dynamic and connected to the database, follow these steps:

### 1. Start the Backend Server
Open your terminal in the `system-backend` directory and run:
```bash
cd system-backend
npm install  # (First time only)
npm start
```
*The server will run on `http://localhost:3001`.*

### 2. Open the Previews
Once the server is running, open the following files in your browser:
- **Admin Portal**: `preview-web-admin.html`
- **Field Rescuer App**: `preview-rescuer.html`
- **Public SOS App**: `preview-mobile-app.html`

*Note: All apps are configured to communicate with `localhost:3001` automatically.*

## 📂 Database & Seed Scripts
If you ever need to reset or re-seed the dummy data, use these scripts in the `system-backend` folder:
- `node seed_data.js`: Basic user seeding.
- `node seed_groups.js`: Advanced group and rescuer seeding.
- `node hard_reset.js`: **Recommended** - Wipes the database and re-initializes it with exactly 2 teams and 9 users (as per this guide).

---

## 📝 Future Modifications
- **Adding Users**: Use the `Member Master DB` in the **Web Admin Portal** to add new rescuers or citizens dynamically.
- **API URL**: If you deploy the backend to a cloud server, update the `API_BASE` or `this.API` variables in the `.html` files to point to your new server URL.
